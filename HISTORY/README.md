# History

History stores chronological records of meaningful Codex work.

Use History to record what was requested, what changed, what was inspected, how work was validated, and what risks remain.

## History Files

| File | Use |
| --- | --- |
| [TASK_HISTORY.md](TASK_HISTORY.md) | High-level chronological task summaries. |
| [CODEX_RUN_LOG.md](CODEX_RUN_LOG.md) | Detailed Codex run records. |
| [DEPLOYMENT_LOG.md](DEPLOYMENT_LOG.md) | Deployment and validation command records. |
| [TEST_RESULTS_LOG.md](TEST_RESULTS_LOG.md) | Apex test command and result records. |
| [CHANGE_ARCHIVE.md](CHANGE_ARCHIVE.md) | Larger completed change summaries. |

## What Belongs In History

- User request summaries.
- Files inspected.
- Files changed.
- Validation commands and results.
- Deployment and test outcomes.
- Residual risks and limits.
- Links to workspace reports when useful.

## What Does Not Belong In History

- Credentials or auth data.
- Private customer data.
- Raw private logs.
- Local-only paths.
- Unnecessary full command dumps.
- Reusable lessons that belong in Memory.

## Update Rule

After meaningful work, add a concise chronological entry.

History should be factual, public-safe, and easy to scan.
