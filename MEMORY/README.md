# Memory

Memory stores reusable knowledge Codex should carry into future tasks.

Use Memory for durable lessons, stable decisions, verified project patterns, and facts that remain useful beyond one run.

## Memory Files

| File | Use |
| --- | --- |
| [PROJECT_MEMORY.md](PROJECT_MEMORY.md) | Stable repo facts and durable lessons. |
| [FIX_HISTORY.md](FIX_HISTORY.md) | Reusable fix patterns and lessons learned. |
| [FAILURE_HISTORY.md](FAILURE_HISTORY.md) | Failure causes and prevention rules. |
| [DECISION_LOG.md](DECISION_LOG.md) | Architecture and documentation decisions. |
| [PROJECT_NOTES.md](PROJECT_NOTES.md) | Stable project notes and assumptions. |
| [KNOWN_ORG_PATTERNS.md](KNOWN_ORG_PATTERNS.md) | Verified object, field, folder, and project-specific Salesforce patterns. |

## What Belongs In Memory

- Confirmed reusable fix patterns.
- Stable repo facts.
- Durable architecture decisions.
- Verified Salesforce object, field, metadata, and folder patterns.
- Lessons that should guide future Codex work.

## What Does Not Belong In Memory

- Credentials or tokens.
- Private customer data.
- Raw org exports.
- Private debug logs.
- Local-only paths.
- One-off command output.
- Unverified guesses.

## Update Rule

After meaningful work, update Memory only when there is a reusable lesson or stable fact.

Use History for chronological task records.
