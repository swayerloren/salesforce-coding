# Salesforce Coding v1.0.0

Initial public release of `salesforce-coding`.

This release publishes a public Salesforce coding knowledge base for Codex, AI coding agents, and human Salesforce developers.

## Included Knowledge Areas

- Apex services, triggers, controllers, async work, dynamic SOQL, DML, and tests.
- LWC templates, getters, navigation, toast, refresh, record pages, mobile layouts, and modal behavior.
- Salesforce metadata structure, FlexiPages, Dynamic Forms, quick actions, layouts, and activation pitfalls.
- Deployment runbooks, test selection guidance, validation habits, and common deployment failures.
- Salesforce Files behavior, including `ContentVersion`, `ContentDocument`, preview, download, latest versions, rollups, and mobile quirks.
- Visualforce PDF maintenance and Aura template-region guidance.
- Platform limitations that should shape implementation decisions.

## Main Sections

- Root Salesforce guides by topic.
- `patterns/` for reusable implementation patterns.
- `examples/` for generic Apex, LWC, and metadata examples.
- `troubleshooting/` for failure-specific guides.
- `prompts/` for Codex and AI-agent workflows.
- `checklists/` for preflight reviews.
- `anti_patterns/` for repeatedly failing approaches.

## Intended Audience

- Salesforce developers.
- Salesforce admins and architects.
- Code reviewers.
- Teams using Codex or other AI agents for Salesforce work.

## Public-Safety Note

The knowledge in this release is distilled from real Salesforce implementation and debugging experience. Private customer data, org-specific details, internal URLs, credentials, usernames, deploy ids, record ids, and sensitive examples were removed or generalized before publication.
