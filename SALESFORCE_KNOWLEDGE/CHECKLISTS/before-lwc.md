# Before Writing LWC Checklist

- [ ] Read `.js-meta.xml`.
- [ ] Confirm targets and form factors.
- [ ] Confirm direct placement or parent/child role.
- [ ] Decide LDS/UI API vs Apex.
- [ ] Decide wire vs imperative Apex and cache behavior.
- [ ] Decide `lightning-record-edit-form` vs custom form for record editing.
- [ ] Verify dynamic field metadata, access, and compound-field behavior if present.
- [ ] Plan loading, empty, error, and success states.
- [ ] Plan refresh after mutation.
- [ ] Avoid template expressions.
- [ ] Use supported NavigationMixin PageReferences.
- [ ] Use ShowToastEvent only after meaningful completion.
- [ ] Check desktop and mobile layout.
- [ ] Check quick action/mobile action bar visibility when actions are involved.
- [ ] Check file upload, preview, download, latest-version, and generated PDF behavior when files are involved.
- [ ] Use theme-safe colors.
