# Mobile Compatibility Checklist

- [ ] Component supports Small form factor if needed.
- [ ] Buttons have touch-safe hit areas.
- [ ] Menus handle blur before click.
- [ ] Overlay has explicit host, shell, scroll lock, and cleanup ownership.
- [ ] Fullscreen elements use stable viewport sizing.
- [ ] File/camera/PDF paths are tested in Salesforce mobile.
- [ ] Popup and navigation fallbacks exist.
- [ ] Close restores page interaction.
- [ ] Debug state is visible when console logs are inaccessible.
