# Sanitized Lessons Catalog

## Lessons From Repeated Failures

- Production deploys need test strategy even for UI metadata.
- Selected-class coverage can block a deploy after tests pass.
- File-heavy tests need special care because Salesforce Files can row-lock.
- Mobile webview failures need visible state, not only terminal reasoning.
- LWC template expressions must be getters.
- FlexiPage assignment must be verified by form factor.
- Standard and custom related lists refresh differently.
- Visualforce PDFs value stable output over modern markup.
- Compound address fields require leaf-field handling.
- UI-only duplicate prevention is not enough.
- Optional dynamic Apex calls can hide missing behavior.
