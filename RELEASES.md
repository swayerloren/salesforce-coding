# Releases

## Release Strategy

This repository uses semantic versioning for public documentation releases.

- `v1.0.0`: initial public foundation.
- Patch releases: documentation fixes, broken links, typo fixes, small clarifications, and public-safety corrections.
- Minor releases: new Salesforce guide sections, examples, prompts, checklists, or troubleshooting categories.
- Major releases: large structural reorganizations or changes that make old links or usage patterns obsolete.

## v1.0.0

Release title: Salesforce Coding v1.0.0

Release purpose:

- Publish the initial public Salesforce coding knowledge base.
- Provide reusable guidance for Codex and human Salesforce developers.
- Make LWC, Apex, metadata, deployment, testing, debugging, files, mobile, Aura, Visualforce, and record-page lessons easy to find.
