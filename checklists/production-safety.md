# Production Safety Checklist

- [ ] Change is minimal and targeted.
- [ ] No unrelated refactor.
- [ ] Required behavior fails loudly.
- [ ] Optional behavior is documented.
- [ ] No private data or secrets added.
- [ ] Tests cover changed behavior.
- [ ] Deployment package is narrow.
- [ ] Rollback or mitigation path is understood.
- [ ] User-facing errors are clear.
- [ ] Logs/debug output do not leak sensitive values.
