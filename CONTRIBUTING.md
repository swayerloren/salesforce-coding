# Contributing

Contributions are welcome when they improve reusable Salesforce engineering guidance and remain safe for a public repository.

## Before You Open A Pull Request

- Read [SALESFORCE_KNOWLEDGE/DOCS/public-sanitization-policy.md](SALESFORCE_KNOWLEDGE/DOCS/public-sanitization-policy.md).
- Review [CONTRIBUTING_GUIDELINES.md](CONTRIBUTING_GUIDELINES.md).
- Remove private org names, customer names, emails, URLs, record ids, deploy ids, screenshots, tokens, credentials, and internal-only notes.
- Prefer reusable Salesforce lessons over one-off org history.
- Keep examples generic and copy-paste-safe.

## Good Contributions

- Focused Apex, LWC, metadata, deployment, testing, mobile, file, or debugging lessons.
- Reusable examples with generic objects and fields.
- Troubleshooting entries with symptom, root cause, and fix.
- Checklists that prevent deployment or production failures.
- Prompts that help Codex or other AI agents work safely in Salesforce repos.

## Pull Request Checklist

- [ ] Content is public-safe.
- [ ] Examples use generic names and no real identifiers.
- [ ] Links are public documentation or repo-local links.
- [ ] Guidance explains why the pattern works.
- [ ] Markdown links and headings are readable.
- [ ] No generated junk, logs, or archives are committed.
